---

## 📁 Структура репозитория

- `practice-1/` — CSS-препроцессоры (SASS)
- `practice-2/` — Сервер на Node.js + Express
- `practice-3/` — JSON и тестирование API в Postman
- `practice-4/` — React + API
- `practice-5/` — Расширенный REST API + Swagger
- `final-project/` — **Итоговый проект** (готовое приложение)
- `README.md` — **Финальный отчёт** по всему курсу

### Использованные технологии и их применение

| Технология                  | Где применена                          | Для чего использовалась |
|----------------------------|----------------------------------------|-------------------------|
| **SASS**                   | `practice-1`, `final-project/frontend` | Переменные, миксины, вложенность, условные конструкции, тёмная тема |
| **Express.js**             | `practice-2`, `practice-4`, `practice-5`, `final-project/backend` | Создание REST API, обработка CRUD-запросов |
| **Node.js**                | Все backend-части                      | Серверная среда выполнения |
| **Postman**                | `practice-3`                           | Тестирование API (GET, POST, PATCH, DELETE) |
| **React**                  | `practice-4`, `final-project/frontend` | Интерактивный пользовательский интерфейс |
| **Fetch API**              | `final-project/frontend`               | Взаимодействие фронтенда с бэкендом |
| **Swagger (OpenAPI)**      | `practice-5`, `final-project/backend`  | Автоматическая генерация документации API |
| **nanoid**                 | `final-project/backend`                | Генерация уникальных ID товаров |
| **CORS**                   | `final-project/backend`                | Разрешение запросов с фронтенда |
| **localStorage**           | `final-project/frontend`               | Сохранение корзины |

### Реализованный функционал финального проекта
- Полноценный CRUD для товаров
- Добавление товаров с возможностью указать фото (или рандомное фото)
- Корзина с подсчётом количества и суммы
- Редактирование и удаление товаров
- Современный адаптивный дизайн с анимациями
- Тёмная тема
- Интерактивная документация Swagger

В ходе выполнения практических работ и финального проекта я освоил полный цикл разработки веб-приложения: от стилизации с помощью препроцессоров до создания REST API и документирования его с помощью Swagger. Полученные знания позволяют создавать современные полнофункциональные веб-приложения.
